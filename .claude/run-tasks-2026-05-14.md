### #17 — Build + install 6 TTE monitoring checks + systemd timer

**What**: Create 6 scripts at /opt/stockbuddy/monitoring/checks/tte-*.sh. Each:
- Runs the check (Mongo query OR docker logs grep)
- Exits 0 if healthy
- If unhealthy: POSTs JSON alert to http://127.0.0.1:8765/alert AND exits 1
- Dedupes via state file at /opt/stockbuddy/cc-state/check-state/<name>.json with min_interval_between_alerts=1h

The 6 checks:
1. `tte-snapshot-success-rate.sh` — Mongo: %completed / total in last 1h. Alert if <80 % AND total ≥ 10 (avoid noise during quiet markets).
2. `tte-snapshot-failed-burst.sh` — Mongo: count snapshotStatus=failed in last 30 min. Alert if >10.
3. `tte-change-symbol-errors.sh` — `docker logs tte-1 --since 30m | grep -c "Failed to change symbol"`. Alert if >5.
4. `tte-alert-sidebar-errors.sh` — `docker logs tte-1 --since 30m | grep -cE "alerts sidebar|tabpanel_log"`. Alert if >5.
5. `tte-container-restart.sh` — `docker inspect tte-1 --format '{{.RestartCount}}'` delta vs last check. Alert if Δ > 0.
6. `tte-webhook-delivery.sh` — During US/IN market hours: `docker logs tte-1 --since 30m | grep -c "Webhook sent.*200"`. Alert if 0.

Systemd timer `tte-checks.timer` at /etc/systemd/system/, firing every 5 min, calling `/opt/stockbuddy/monitoring/run-checks.sh` (which iterates the 6 scripts).

**Acceptance criteria**:
- AC17.1: All 6 scripts exist + are executable + stockbuddy-owned.
- AC17.2: `systemctl status tte-checks.timer` shows Active.
- AC17.3: Manual `bash /opt/stockbuddy/monitoring/checks/tte-snapshot-success-rate.sh` exits 0 on currently-healthy state.
