# Spec: Reversed-Strategy Snapshots

## Goal
Make TradingView snapshots generated by TTE visually reflect the **reversed** strategy used by Stock Buddy on read.

- **Goal 1 (forward)**: Every newly-generated snapshot draws TP where the original SL would be, and SL where the original TP would be. Target visual R:R = 2:1 (reversed from the original 1:2).
- **Goal 2 (backfill)**: Re-render every existing snapshot in Stock Buddy's `setup_messages` collection so historical visuals also show the reversed levels. Long-running job, may take up to 1 day.

The webhook payload sent to Stock Buddy is unchanged — Stock Buddy already flips on read. Only what TradingView **draws** changes.

## Acceptance criteria

### Goal 1 (forward snapshots)
- [ ] TP coordinate equals the original-strategy SL price; SL coordinate equals the original-strategy TP price.
- [ ] Resulting visual R:R is 2:1 (reversed) — verified on 3 sample symbols (1 crypto, 1 forex, 1 stock).
- [ ] Behaviour gated by env flag `REVERSED_STRATEGY_SNAPSHOTS` (default `true`); flipping to `false` restores original-strategy drawing for rollback.
- [ ] Setup data sent via webhook to Stock Buddy is byte-identical to pre-change baseline (no schema or payload regression).
- [ ] No change to maintenance loop / alert creation behaviour.

### Goal 2 (backfill)
- [ ] Script enumerates **all** docs in `setup_messages` (Stock Buddy MongoDB), regenerates the snapshot via TTE, updates the doc's image field.
- [ ] **Idempotent**: re-running on an already-processed doc is a no-op (or safely re-renders with the same result).
- [ ] **Checkpointable**: progress persisted to a local file (e.g. `.claude/backfill-progress.json`) so a crash/restart resumes from the last completed `_id`.
- [ ] Progress logged every 30 min to `.claude/team-bus-C.md` as `[BACKFILL] processed N / M`.
- [ ] Backfill failure on an individual doc does not abort the run — error logged, doc skipped, retry list emitted at end.

## Scope
- Forward: snapshot drawing code in TTE (Selenium line-tool clicks for TP/SL).
- Backfill: standalone script in TTE repo that drives the same drawing path against historical setups.
- Stock Buddy: **read-only** for the backfill (read setup data, write back regenerated image URL only).

## Out of scope
- Stock Buddy schema changes (none needed — flip-on-read already in place).
- Pine Script changes.
- Alert creation / maintenance logic.
- Webhook payload structure.
