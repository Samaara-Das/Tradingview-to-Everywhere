# Point Capital Project - Product Requirements Document

## Project Overview

Transform the existing TradingView to Everywhere (TTE) application to work with 3 new Pine Script screener indicators instead of the current Premium Screener system. The goal is to capture alert data and screenshots from these new screeners and store them in a MongoDB database, while maintaining the core automation framework.

## Background

The current TTE system uses Premium Screener, Trade Drawer, and Get Exits indicators to process trading signals from 1300+ symbols across 4 categories. This project replaces the existing screener functionality with 3 new specialized screeners that provide different types of trading signals with unique JSON message formats.

## Goals and Objectives

### Primary Goal
Replace the existing TTE screener system with 3 new Pine Script screener indicators to capture and store their alert data in MongoDB.

### Success Criteria
- Successfully process alerts from all 3 new screeners
- Store alert data with screenshots in new MongoDB collection
- Maintain existing symbol processing (1300+ symbols, 4 categories, 5 symbols per alert)
- Process alerts sequentially as they appear in TradingView logs
- Disable social media distribution initially while preserving code for future use

## Target Users

- Trading signal analysts who need automated data collection from multiple screener indicators
- System operators who configure and monitor the automated trading signal processing

## Data Models

### Alert Data Structure
Each processed alert should include:
- Screener source identifier (Order Block/Structure Break/Nadaraya Watson)
- Original JSON message data (varies by screener)
- Symbol and timeframe information
- Direction/signal type
- Timestamp (parsed and standardized)
- Screenshot links (PNG and TradingView)
- Processing metadata

### Screener-Specific Fields

#### Structure Break Screener
- Type (t): "sb"
- Symbol (s): Trading symbol
- Timeframe (tf): Chart timeframe
- Direction (d): "bullish" or "bearish"
- ZigZag degree (zz): "ZZ1", "ZZ2", "ZZ3"
- Timestamp (ts): Alert timestamp
- Price (p): Current price

#### Nadaraya Watson Screener
- Type: "nw"
- Symbol: Trading pair
- Timeframe: Time period
- Direction: "buy" or "sell"
- Timestamp: UTC timestamp

#### Order Block Screener
- Timestamp: ISO format
- Symbol: Trading symbol
- Timeframe: Timeframe string
- Signal_type: "buy_signal" or "sell_signal"
- Ob_type: "bullish" or "bearish"
- Ob_high: Order block high price
- Ob_low: Order block low price
- Current_price: Current market price
